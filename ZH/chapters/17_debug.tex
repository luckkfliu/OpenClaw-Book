\section{Node + tsx "\textbackslash\{\}\_\textbackslash\{\}\_name is not a function" 崩溃}


\subsection{概述}


通过 Node 使用 \texttt{tsx} 运行 OpenClaw 时，启动阶段报错：

\begin{lstlisting}
[openclaw] Failed to start CLI: TypeError: __name is not a function
    at createSubsystemLogger (.../src/logging/subsystem.ts:203:25)
    at .../src/agents/auth-profiles/constants.ts:25:20
\end{lstlisting}


此问题在开发脚本从 Bun 切换到 \texttt{tsx} 后出现（提交 \texttt{2871657e}，2026-01-06）。相同的运行路径在 Bun 下正常工作。

\subsection{环境}


\begin{itemize}
  \item Node: v25.x（在 v25.3.0 上观察到）
  \item tsx: 4.21.0
  \item 操作系统: macOS（其他运行 Node 25 的平台也可能复现）
\end{itemize}


\subsection{复现步骤（仅 Node）}


\begin{lstlisting}[language=bash]
# 在仓库根目录
node --version
pnpm install
node --import tsx src/entry.ts status
\end{lstlisting}


\subsection{仓库内最小复现}


\begin{lstlisting}[language=bash]
node --import tsx scripts/repro/tsx-name-repro.ts
\end{lstlisting}


\subsection{Node 版本检查}


\begin{itemize}
  \item Node 25.3.0：失败
  \item Node 22.22.0（Homebrew \texttt{node@22}）：失败
  \item Node 24：尚未安装，需要验证
\end{itemize}


\subsection{说明 / 假设}


\begin{itemize}
  \item \texttt{tsx} 使用 esbuild 转换 TS/ESM。esbuild 的 \texttt{keepNames} 会生成一个 \texttt{\_\_name} 辅助函数，并用 \texttt{\_\_name(...)} 包裹函数定义。
  \item 崩溃表明 \texttt{\_\_name} 存在但在运行时不是函数，这意味着在 Node 25 的加载器路径中该辅助函数缺失或被覆盖。
  \item 其他 esbuild 使用者也报告过类似的 \texttt{\_\_name} 辅助函数缺失或被重写的问题。
\end{itemize}


\subsection{回归历史}


\begin{itemize}
  \item \texttt{2871657e}（2026-01-06）：脚本从 Bun 改为 tsx，使 Bun 成为可选项。
  \item 在此之前（Bun 路径），\texttt{openclaw status} 和 \texttt{gateway:watch} 均正常工作。
\end{itemize}


\subsection{解决方法}


\begin{itemize}
  \item 开发脚本使用 Bun（当前临时回退方案）。
  \item 使用 Node + tsc watch，然后运行编译产物：
\begin{lstlisting}[language=bash]
  pnpm exec tsc --watch --preserveWatchOutput
  node --watch openclaw.mjs status
\end{lstlisting}

  \item 已在本地确认：\texttt{pnpm exec tsc -p tsconfig.json} + \texttt{node openclaw.mjs status} 在 Node 25 上可正常运行。
  \item 如果可能，在 TS 加载器中禁用 esbuild 的 keepNames（防止插入 \texttt{\_\_name} 辅助函数）；tsx 目前不提供此配置项。
  \item 在 Node LTS（22/24）上测试 \texttt{tsx}，确认该问题是否为 Node 25 特有。
\end{itemize}
